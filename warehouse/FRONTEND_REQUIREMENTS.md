# Требования к React фронтенду для системы управления складом

## Архитектура

Приложение будет построено на React 18+ с TypeScript, Ant Design для UI компонентов, React Router для навигации, и axios для HTTP запросов. Структура проекта будет модульной с разделением на features (users, warehouses, cargo).

## Основные компоненты

### 1. Настройка проекта и инфраструктура

- **Инициализация проекта**: Создать React приложение с TypeScript (Vite или Create React App)

- **Установка зависимостей**:
  - `antd` - UI компоненты
  - `@ant-design/icons` - иконки
  - `react-router-dom` - роутинг
  - `axios` - HTTP клиент
  - `react-query` или `@tanstack/react-query` - управление состоянием сервера и кеширование
  - `dayjs` - работа с датами
  - `zod` или `yup` - валидация форм
  - `js-cookie` или `react-cookie` - работа с cookies для хранения токенов (опционально, можно использовать localStorage)

- **Настройка структуры папок**:

  ```text
  src/
    ├── api/           # API клиенты и типы
    ├── components/    # Переиспользуемые компоненты
    ├── features/      # Модули по доменам (users, warehouses, cargo)
    ├── hooks/         # Кастомные хуки
    ├── utils/         # Утилиты
    ├── types/         # Общие типы
    └── App.tsx
  ```

### 2. API слой

- **Конфигурация axios**: Базовый URL, interceptors для обработки ошибок и автоматического добавления токена авторизации в заголовки запросов

- **Типы TypeScript**: Определить интерфейсы для всех сущностей (User, Warehouse, Cargo, PaginatedResponse, AuthResponse, LoginDto, RegisterDto)

- **API клиенты**: Создать модули для каждого ресурса:
  - `authApi.ts` - методы для авторизации и регистрации (login, register, logout)
  - `usersApi.ts` - методы для работы с пользователями
  - `warehousesApi.ts` - методы для работы со складами
  - `cargoApi.ts` - методы для работы с грузами

- **Хуки для React Query**: Создать кастомные хуки для каждого API метода (useLogin, useRegister, useUsers, useCreateUser, useUpdateUser, etc.)

- **Управление аутентификацией**:
  - Создать контекст или хук для управления состоянием авторизации (AuthContext/AuthProvider)
  - Хранение токена в localStorage или cookies
  - Функции для проверки авторизации и получения текущего пользователя
  - Автоматическое добавление токена в заголовок Authorization для всех запросов

### 3. Общие компоненты

- **Layout**: Основной layout с навигацией (меню, хедер, сайдбар)

- **Pagination**: Компонент пагинации на базе Ant Design Pagination

- **Filters**: Универсальный компонент фильтров с поддержкой:
  - Текстовых полей (Input)
  - Числовых диапазонов (InputNumber с Min/Max)
  - Дат (DatePicker с диапазоном)
  - Селектов (Select для enum и boolean)

- **Table**: Обертка над Ant Design Table с поддержкой пагинации и фильтрации

- **FormModal**: Модальное окно для создания/редактирования с валидацией

- **ErrorBoundary**: Обработка ошибок React

- **Loading**: Компоненты загрузки (Spin, Skeleton)

- **ProtectedRoute**: Компонент для защиты маршрутов, требующих авторизации

### 4. Модуль авторизации и регистрации

- **Страница входа** (`LoginPage.tsx`):
  - Форма с полями: email, password
  - Валидация: email должен быть валидным, password минимум 6 символов
  - Кнопка "Войти"
  - Ссылка на страницу регистрации
  - Обработка ошибок авторизации (неверный email/пароль, неактивный аккаунт)
  - После успешного входа: сохранение токена и данных пользователя, редирект на главную страницу

- **Страница регистрации** (`RegisterPage.tsx`):
  - Форма с полями: email, password, confirmPassword
  - Валидация:
    - email должен быть валидным
    - password минимум 6 символов
    - confirmPassword должен совпадать с password
  - Кнопка "Зарегистрироваться"
  - Ссылка на страницу входа
  - Обработка ошибок регистрации (email уже существует)
  - После успешной регистрации: автоматический вход, сохранение токена и данных пользователя, редирект на главную страницу

- **Компонент выхода** (`LogoutButton.tsx`):
  - Кнопка выхода в навигации/хедере
  - При клике: очистка токена и данных пользователя, редирект на страницу входа
  - Подтверждение выхода (опционально)

- **Отображение информации о пользователе**:
  - В хедере/навигации отображать email текущего пользователя
  - Показывать роль пользователя (если нужно)

### 5. Модуль управления пользователями

- **Список пользователей** (`UsersList.tsx`):
  - Таблица с колонками: ID, Email, ФИО, Роль, Активность, Дата создания
  - Фильтры: email, firstName, lastName, middleName, position, department, role, isActive, даты создания/обновления
  - Пагинация
  - Кнопки действий: Создать, Редактировать, Удалить

- **Форма создания/редактирования пользователя** (`UserForm.tsx`):
  - Поля: email, password (только при создании), firstName, lastName, middleName, position, department, permission, role, isActive
  - Валидация всех полей согласно API требованиям

- **Детальная страница пользователя** (`UserDetail.tsx`): Просмотр полной информации о пользователе

### 6. Модуль управления складами

- **Список складов** (`WarehousesList.tsx`):
  - Таблица с колонками: ID, Название, Размеры (X×Y×Z), Описание, Количество грузов, Дата создания
  - Фильтры: name, description, размеры (xMin/xMax, yMin/yMax, zMin/zMax), даты
  - Пагинация
  - Кнопки действий: Создать, Редактировать, Удалить, Просмотр склада (2D визуализация)

- **Форма создания/редактирования склада** (`WarehouseForm.tsx`):
  - Поля: x, y, z (размеры), name, description
  - Валидация размеров (>= 1)

- **Детальная страница склада** (`WarehouseDetail.tsx`):
  - Информация о складе
  - Список грузов на складе
  - 2D визуализация склада

### 7. Модуль управления грузами

- **Список грузов склада** (`CargoList.tsx`):
  - Таблица с колонками: ID, GUID, Тип, Название, Размеры, Вес, Позиция (X,Y,Z), Дата хранения
  - Фильтры: guid, type, name, description, веса (netWeight, packWeight, totalWeight, maxLoadWeight), размеры, позиция, transportNumber, storageDate
  - Пагинация
  - Кнопки действий: Добавить, Редактировать, Переместить, Удалить

- **Форма добавления груза** (`CargoForm.tsx`):
  - Поля: type (radio/select), guid, name, description, weight, sizeX/Y/Z, containSizeX/Y/Z (для контейнеров), transportNumber, storageDate, позиция (x, y, z)
  - Условная валидация для контейнеров (containSize обязателен)

- **Форма редактирования груза** (`CargoEditForm.tsx`):
  - Все поля кроме позиции (позиция изменяется через отдельную форму перемещения)

- **Форма перемещения груза** (`MoveCargoForm.tsx`):
  - Поля: x, y, z (новая позиция)
  - Валидация позиции (>= 0, проверка границ склада)

### 8. 2D визуализация склада

- **Компонент WarehouseVisualization** (`WarehouseVisualization.tsx`):
  - Canvas или SVG для отрисовки склада
  - Отображение склада как прямоугольника с размерами
  - Отображение грузов как прямоугольников на их позициях
  - Цветовое кодирование: разные цвета для контейнеров и грузов
  - Интерактивность:
    - Hover для показа информации о грузе
    - Клик для открытия детальной информации
    - Возможность перетаскивания грузов (drag & drop) для перемещения
  - Легенда с типами грузов
  - Масштабирование и панорамирование (zoom, pan)

- **Утилиты для расчета**:
  - Функция проверки пересечений грузов
  - Функция проверки границ склада
  - Расчет оптимального размещения

### 9. Роутинг

- Настроить React Router с маршрутами:
  - `/login` - страница входа (публичный маршрут)
  - `/register` - страница регистрации (публичный маршрут)
  - `/` - главная страница (дашборд, защищенный маршрут)
  - `/users` - список пользователей (защищенный маршрут)
  - `/users/:id` - детали пользователя (защищенный маршрут)
  - `/warehouses` - список складов (защищенный маршрут)
  - `/warehouses/:id` - детали склада с визуализацией (защищенный маршрут)
  - `/warehouses/:id/cargo` - список грузов склада (защищенный маршрут)
  - `/warehouses/:id/cargo/:cargoId` - детали груза (защищенный маршрут)

- **Защита маршрутов**:
  - Использовать компонент ProtectedRoute для всех маршрутов, требующих авторизации
  - При попытке доступа к защищенному маршруту без авторизации - редирект на `/login`
  - После успешного входа - редирект на изначально запрошенную страницу или на главную
  - При попытке доступа к `/login` или `/register` для авторизованного пользователя - редирект на главную страницу

### 10. Обработка ошибок и валидация

- **Глобальная обработка ошибок**: Interceptor для axios, отображение ошибок через Ant Design message/notification

- **Обработка ошибок авторизации**:
  - При получении 401 (Unauthorized) - автоматический выход пользователя, очистка токена, редирект на страницу входа
  - Отображение понятных сообщений об ошибках авторизации (неверный email/пароль, неактивный аккаунт)

- **Валидация форм**: Использовать Ant Design Form с правилами валидации

- **Обработка ошибок валидации API**: Парсинг ответов 400 и отображение ошибок в формах

### 11. Дополнительные функции

- **Экспорт данных**: Кнопки экспорта таблиц в CSV/Excel (опционально)

- **Поиск**: Глобальный поиск по всем ресурсам (опционально)

- **Уведомления**: Toast уведомления для успешных операций

- **Подтверждение удаления**: Модальные окна подтверждения перед удалением

## Технические детали

- **State Management**:
  - React Query для серверного состояния
  - React Context для состояния аутентификации (AuthContext)
  - Zustand или другой state manager для клиентского состояния (если нужно)

- **Стилизация**: Ant Design темизация, кастомные стили через CSS modules или styled-components при необходимости

- **Производительность**:
  - Мемоизация компонентов (React.memo)
  - Виртуализация длинных списков (react-window, если нужно)
  - Lazy loading для роутов

- **Доступность**: Использовать семантические компоненты Ant Design, ARIA атрибуты

## Тестирование (опционально)

- Unit тесты для утилит и хуков (Jest + React Testing Library)
- Integration тесты для форм и таблиц
- E2E тесты для критических сценариев (Playwright или Cypress)

## Список задач

1. Инициализировать React проект с TypeScript, установить зависимости (antd, react-router-dom, axios, react-query, dayjs, js-cookie), настроить структуру папок

2. Создать API слой: конфигурация axios с interceptors для токенов, типы TypeScript для всех сущностей (включая AuthResponse), API клиенты (authApi, usersApi, warehousesApi, cargoApi), React Query хуки

3. Реализовать модуль авторизации: AuthContext/AuthProvider для управления состоянием, функции для работы с токеном (сохранение, получение, удаление), компоненты LoginPage и RegisterPage с валидацией, обработка ошибок авторизации

4. Создать общие компоненты: Layout с навигацией (включая отображение пользователя и кнопку выхода), ProtectedRoute для защиты маршрутов, Pagination, Filters (универсальный компонент), Table обертка, FormModal, ErrorBoundary, Loading компоненты

5. Настроить React Router с маршрутами: публичные маршруты (/login, /register), защищенные маршруты (/, /users, /warehouses, /cargo), логика редиректов для авторизованных/неавторизованных пользователей

6. Реализовать модуль пользователей: UsersList (таблица с фильтрами), UserForm (создание/редактирование), UserDetail (детальная страница)

7. Реализовать модуль складов: WarehousesList (таблица с фильтрами), WarehouseForm (создание/редактирование), WarehouseDetail (детальная страница)

8. Реализовать модуль грузов: CargoList (таблица с фильтрами), CargoForm (добавление), CargoEditForm (редактирование), MoveCargoForm (перемещение)

9. Реализовать 2D визуализацию склада: WarehouseVisualization компонент (Canvas/SVG), отображение склада и грузов, интерактивность (hover, click, drag & drop), масштабирование и панорамирование

10. Реализовать обработку ошибок: глобальные interceptors (включая обработку 401), валидация форм, отображение ошибок API, уведомления об успешных операциях

11. Добавить финальные улучшения: подтверждение удаления, оптимизация производительности (мемоизация, lazy loading), темизация Ant Design
